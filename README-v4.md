# 🚀 CWP Installer v4.0 - SSH-Safe Edition

## ✨ ما الجديد في v4.0

تم حل مشكلة قطع SSH بالكامل! السكربت الآن:

| المشكلة القديمة | الحل في v4.0 |
|----------------|--------------|
| ❌ SSH يقطع الاتصال أثناء التحديث | ✅ يطلق نفسه في `screen` تلقائياً |
| ❌ يتوقف التركيب عند قطع الاتصال | ✅ يستمر العمل في الخلفية حتى بعد قطع SSH |
| ❌ لا توجد إشارة لمعرفة التقدم | ✅ شريط تقدم تفاعلي + 22 مرحلة |
| ❌ لا يمكن متابعة التركيب من اتصال جديد | ✅ `--monitor` لمتابعة حية من أي اتصال |
| ❌ SSH timeout أثناء العمليات الطويلة | ✅ Keep-alive 60 ثانية تلقائياً |
| ❌ ضياع التقدم عند قطع الاتصال | ✅ حفظ الحالة في `/var/lib/cwp-installer/` |

## 🎯 الاستخدام (3 خطوات فقط)

### الخطوة 1: رفع السكربت
```bash
scp cwp-installer-v4.sh root@SERVER_IP:/root/
```

### الخطوة 2: تعديل الإعدادات
```bash
ssh root@SERVER_IP
nano cwp-installer-v4.sh
```

**عدّل القسم `1) CONFIGURATION`:**
```bash
SERVER_HOSTNAME="server1.example.com"   # FQDN
ADMIN_EMAIL="admin@example.com"
TIMEZONE="Africa/Cairo"
DEFAULT_PHP_VERSION="8.3"
SSH_PORT="2200"
```

### الخطوة 3: التشغيل
```bash
chmod +x cwp-installer-v4.sh
./cwp-installer-v4.sh
```

**هيحصل التالي تلقائياً:**
1. ✅ السكربت هيكتشف إنك في SSH ويُحذرك
2. ✅ هيطلق نفسه داخل `screen` session
3. ✅ هيفعّل SSH keep-alive
4. ✅ هيعرض شريط التقدم

## 🛠️ سيناريوهات الاستخدام

### السيناريو 1: التركيب الطبيعي
```bash
./cwp-installer-v4.sh
# ينتظر 5 ثوانٍ → يطلق screen → يبدأ التركيب
```

### السيناريو 2: انقطع SSH أثناء التركيب
**لا قلق!** التركيب يستمر. للعودة:

```bash
# 1) اعمل SSH تاني للسيرفر
ssh root@SERVER_IP

# 2) ارجع للجلسة
screen -r cwp-installer

# أو شوف التقدم بدون دخول الجلسة
./cwp-installer-v4.sh --monitor
```

### السيناريو 3: متابعة التقدم من اتصال آخر
```bash
./cwp-installer-v4.sh --monitor
```

ستظهر شاشة حية تتحدّث كل 3 ثوانٍ بـ:
- المرحلة الحالية وعدد المراحل المتبقية
- النسبة المئوية + شريط مرئي
- الوقت المنقضي و المتبقي تقريباً
- آخر سطر من السجل
- قائمة كل المراحل (✓ مكتمل، ▶ جاري، ○ منتظر)

### السيناريو 4: عرض الحالة سريعاً
```bash

# المرحلة:  [12/22] تحسين MySQL
# النسبة:   54%
```

### السيناريو 5: متابعة السجل المباشر

## 🔄 آلية الحماية من قطع SSH

السكربت يطبّق طبقات حماية متعددة:

### الطبقة 1: SSH Keep-Alive
```
ClientAliveInterval 60      # يرسل ping كل 60 ثانية
ClientAliveCountMax 720     # حتى 12 ساعة من عدم النشاط
TCPKeepAlive yes            # TCP keepalive نشط
```

### الطبقة 2: Screen Session
```bash
screen -S cwp-installer  # session مسماة تبقى حتى بعد قطع SSH
```

### الطبقة 3: Heartbeat
```bash
# نبض كل 10 ثوانٍ → ملف /var/lib/cwp-installer/heartbeat.txt
# يبيّن في --monitor إن السكربت ما زال حياً
```

### الطبقة 4: State Persistence
```bash
/var/lib/cwp-installer/
├── state.txt          # المرحلة الحالية + الحالة
├── progress.txt       # نسبة التقدم
├── heartbeat.txt      # آخر نبضة
├── start_time.txt     # وقت البدء
├── installer.pid      # PID العملية
└── installer.lock     # منع التشغيل المتوازي
```

## ⚠️ ملاحظات مهمة

### قبل التشغيل
- ✅ تأكد أن `mosh` أو `screen` متاحان (السكربت يحاول تركيب `screen` تلقائياً)
- ✅ لو السيرفر عنده console (KVM/IPMI) خليه مفتوح كـ backup

### إذا فشل التركيب
```bash
# راجع السجل
ls -lt /var/log/cwp-installer-*.log
tail -100 /var/log/cwp-installer-*.log

# نظّف State وحاول مرة أخرى
rm -rf /var/lib/cwp-installer/
./cwp-installer-v4.sh
```

### الفرق بين screen و tmux
- السكربت يفضّل `screen` (أبسط ومركّب افتراضياً)
- لو screen مش متوفر، يجرّب `tmux`
- لو الاتنين مش متوفرين، يستخدم `nohup` (بدون إمكانية العودة للجلسة)
